Commit message规范和Commitizen使用

Git 每次提交代码，都要写 commit message（提交说明），否则就不允许提交。

$ git commit -m "hello world"

虽说理论上讲commit message想怎么写都可以，但是一个规范的项目，message应该应该清晰明了，说明本次提交的目的，多名成员最好共用一套commit规范。

虽然海马汇PC端IM项目只有我一个人T_T，但我在项目初期就尝试使用了一套目前使用最广的Angular 规范，并用了配套的工具Commitizen。用起来效果确实很好，在检查更新历史、生成版本更新说明的时候更有效率。

二.Commitizen

Commitizen是一个撰写合格 Commit message 的工具。
安装命令如下。

npm install -g commitizen

然后，在项目目录里，运行下面的命令，使其支持 Angular 的 Commit message 格式。

commitizen init cz-conventional-changelog --save --save-exact

以后，凡是用到

git commit

命令，一律改为使用

git cz

这时，就会出现选项，用来生成符合格式的 Commit message。

再也不会出现名字缩写或者 fix bug、fix bug again之类的commit message了
